0环境要求

软件最低版本说明
Node.js>= 18ESM 模块支持
npm>= 9包管理
MySQL>= 8.0可选，不装也能跑
验证 Node.js 版本：node -v 应输出 v18.x 或更高

1安装依赖

在项目根目录执行：

# 安装后端依赖 cd backend npm install # 安装前端依赖 cd ../frontend npm install

2配置环境变量

后端配置（可选，不配置也能运行）：

cd backend cp .env.example .env # 编辑 .env 文件 # 填入 DeepSeek API Key（可选） # 填入 MySQL 密码（可选）

.env 配置项

变量必填说明
DEEPSEEK_API_KEY否不填使用 Mock 数据
DB_HOST否不填不持久化历史
DB_PASSWORD否MySQL 密码
PORT否默认 3000
不配置任何环境变量也能运行，所有 UI 功能使用 Mock 数据正常体验

3构建前端

将 Vue 源码编译为生产版本：

cd frontend npm run build

构建产物输出到 frontend/dist/ 目录，包含：

  • index.html - 入口文件
  • assets/ - JS/CSS/字体等静态资源
构建成功标志：输出 "built in X.Xs"，无 ERROR

4启动服务

方式一：开发模式（前后端分离）

# 终端1：启动后端 cd backend npm start # 终端2：启动前端（热更新） cd frontend npm run dev

访问：http://localhost:5173

方式二：生产模式（单服务）

后端自动托管前端 dist 静态文件，只需启动后端：

# 先构建前端 cd frontend && npm run build # 启动后端（自动托管前端） cd ../backend npm start

访问：http://localhost:3000

生产模式下前后端在同一端口（3000），无需 Vite 开发服务器

5初始化数据库（可选）

如需持久化历史记录，执行以下 SQL：

mysql -u root -p < backend/src/db/init.sql

并在 .env 中配置数据库连接信息

不初始化数据库也能运行，模拟功能正常，仅历史记录不持久化

6启动命令汇总

场景命令访问地址
开发模式npm run dev (前端)localhost:5173
开发模式npm start (后端)localhost:3000/api
生产模式npm run build + npm startlocalhost:3000
PM2 守护pm2 start src/app.jslocalhost:3000

PM2 进程守护（推荐生产环境）

# 安装 PM2 npm install -g pm2 # 构建前端 cd frontend && npm run build # 启动后端（守护进程） cd ../backend pm2 start src/app.js --name universe-lab # 查看状态 pm2 status # 停止 pm2 stop universe-lab # 重启 pm2 restart universe-lab

7Nginx 反向代理（可选）

如需域名 + HTTPS，配置 Nginx 反向代理：

server { listen 80; server_name your-domain.com; location / { proxy_pass http://127.0.0.1:3000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_cache_bypass $http_upgrade; } }
# 测试配置 sudo nginx -t # 重载 sudo nginx -s reload

9常见问题

Q: 端口被占用？

# 查看占用进程 netstat -ano | findstr :3000 # 修改端口 # 在 backend/.env 中设置 PORT=3001

Q: 前端构建失败？

清除缓存重新安装：

cd frontend rm -rf node_modules package-lock.json npm install npm run build

Q: API 返回 Mock 数据？

检查 backend/.env 中 DEEPSEEK_API_KEY 是否已配置。未配置时自动使用 Mock 数据，UI 功能完整可体验。

Q: 静态文件 404？

确保已执行 npm run build，且 frontend/dist/ 目录存在。后端会自动托管该目录。
